## 공식 요청서 템플릿 (이메일/메신저용)

**제목: 사이니지 관리 시스템 서버 인프라 지원 요청**

안녕하세요, [담당자 이름]님.

현재 개발 중인 **사이니지 관리 시스템(MVP)** 을 사내 서버로 이전하기 위해 인프라 지원을 요청드립니다.

### 시스템 개요
- 웹 기반 관리자 페이지 + TV/디스플레이 화면으로 구성
- 관리자가 조직도·뉴스·방문자 정보를 등록하면 사이니지 화면에 실시간 반영되는 시스템
- 현재 Supabase(외부 클라우드) 사용 중 → 사내 인프라로 이전 예정

### 필요한 환경 (3가지)

**① 애플리케이션 서버** (Next.js 웹앱 실행용)
- OS: Linux (Ubuntu 22.04 LTS 추천) 또는 Windows Server
- CPU: 2코어 이상
- RAM: 4GB 이상
- 저장공간: 20GB 이상
- 필요 소프트웨어: Node.js 22 LTS, PM2(또는 Docker)
- 외부 접근용 포트: 80(HTTP), 443(HTTPS)

**② 데이터베이스 서버** (PostgreSQL)
- PostgreSQL 16 이상
- 별도 서버 또는 ①번 서버에 함께 설치 가능
- 저장공간: 10GB (초기, 향후 확장 가능하면 좋음)
- 앱 서버에서만 접근 가능하도록 방화벽 설정 필요

**③ 파일 스토리지** (이미지·파일 저장용)
- MinIO(Docker로 설치) 또는 회사 NAS/파일 서버 활용 가능
- 저장공간: 50GB 이상 (이미지, 동영상 파일)
- HTTP로 파일 접근 가능해야 함 (퍼블릭 URL 발급)

### 추가로 확인이 필요한 사항

1. 사내 도메인 또는 서브도메인 발급이 가능한지  
   예) `signage.hubilon.com` 또는 고정 IP 사용
2. SSL 인증서 발급 지원 여부 (Let's Encrypt 사용 가능한지)
3. 서버 접속 방법 (SSH 키 발급, VPN 필요 여부)
4. 배포 방식 선호 여부 (직접 SSH 배포 vs CI/CD 파이프라인 vs Docker)

일정 조율이 필요하시면 편하신 시간에 미팅도 가능합니다.  
감사합니다.

## 담당자 답변에 따른 대응 방향

| 담당자 답변 | 대응 방향 |
|---|---|
| "서버 있음, Docker 사용 가능" | 가장 좋은 경우. Docker Compose로 전체 구성 |
| "서버 있는데 Docker 없음" | Node.js + PostgreSQL 직접 설치로 진행 |
| "서버 없음, 클라우드 써야 함" | AWS/NHN/KT 클라우드 계정 요청으로 방향 전환 |
| "DB는 MySQL만 가능" | 마이그레이션 계획 일부 수정 필요 (MIGRATION.md 참고) |

> 담당자가 **"Docker 됩니다"** 라고 하면 이후 설정이 가장 간단함.

## 백엔드 지식 없이 진행 가능 여부

**가능합니다.** 이유:

- 전환 작업의 실체는 **반복 패턴의 코드 교체** (Supabase 쿼리 → API 호출)
- 패턴이 정해지면 Claude가 파일 단위로 처리 가능
- 개발자는 서버 담당자 소통, 환경변수 세팅, 동작 확인 역할만 하면 됨

**단, 전제 조건:**  
인프라(서버)가 준비된 후부터 단계별 진행 가능.  
서버 담당자의 답변을 받은 시점부터 Claude와 함께 다음 단계를 밟으면 됨.

### 단계별 역할 분담

| 단계 | 개발자가 할 일 | Claude가 할 일 |
|---|---|---|
| 인프라 준비 | 서버 담당자에게 요청·소통 | 요청서 작성, 질문 답변 |
| DB 세팅 | 서버에 PostgreSQL 설치 확인 | 설치 명령어, 설정 파일 제공 |
| 코드 전환 | 환경변수 입력, 브라우저 확인 | 코드 전체 수정 |
| 배포 | 서버에 파일 업로드 | 배포 스크립트 작성 |
| QA | 실제 화면 동작 확인 | 오류 분석 및 수정 |
