# 사이니지 관리 시스템 MVP PRD

## 범위 가정

- `/admin` : 사이니지 콘텐츠를 관리하는 **웹 관리자 앱** (PC~모바일 반응형)
- `/` : 실제 TV/키오스크에 출력되는 **사이니지 디스플레이 화면** (비반응형, 풀스크린)
- 계정 생성: 사용자가 직접 회원가입 (이메일 인증 방식). 최초 가입 시 '편집자' 권한 자동 부여
- **조직도 직원 ≠ 시스템 사용자**: 조직도에 표시되는 '직원'은 슈퍼어드민/콘텐츠어드민이 등록·관리하는 데이터이며, 시스템에 로그인하는 '사용자'는 본인이 회원가입한 별개의 계정임

---

## 핵심 정보

**목적**: 하드코딩으로 운영되던 사이니지 콘텐츠를 비개발자도 웹 인터페이스로 자율 편집하고, 변경 사항이 실시간으로 디스플레이 화면에 반영되도록 합니다.
**사용자**: 회사 내부 운영 담당자 — 슈퍼어드민, 콘텐츠어드민, 편집자 3개 역할로 구분됩니다.

---

## 역할 체계

| 역할 | 코드 | 권한 요약 |
|------|------|----------|
| 슈퍼어드민 | `super_admin` | 전체 권한 + 사용자 역할 변경/계정 비활성화 |
| 콘텐츠어드민 | `content_admin` | 모든 콘텐츠 등록/수정/삭제 |
| 편집자 | `editor` | 뉴스·방문자 등록 + 본인 등록 콘텐츠만 수정/삭제 |

## 사용자 여정

```
[관리자 앱 /admin]

1. 비로그인 상태로 /admin 진입
   ↓ 자동 리디렉션

2. 로그인 페이지
   ├── 로그인 버튼 클릭 → 인증 성공 → 대시보드 페이지
   ├── 로그인 실패 → 에러 메시지 표시 (로그인 페이지 유지)
   └── 회원가입 링크 클릭
       ↓

3. 회원가입 페이지
   ↓ 회원가입 제출 → 이메일 인증 메일 발송
   ↓

4. 이메일 인증 완료 페이지 (인증 메일 링크 클릭 시 진입)
   ↓ 인증 완료 → 편집자 권한 자동 부여 → 로그인 페이지로 이동
   ↓

5. 대시보드 페이지 (콘텐츠 현황 개요)
   ↓ 사이드바 메뉴 선택

6. 콘텐츠 관리 분기
   ├── [super_admin, content_admin] 조직도 관리 페이지
   │     → 직원/실(Division)/팀(Team) 등록·수정·삭제
   ├── [전 역할] 뉴스 관리 페이지
   │     → 목록 조회 → 등록/수정/삭제
   │       (editor: 본인 등록 콘텐츠만 수정/삭제)
   ├── [전 역할] 방문자 관리 페이지
   │     → 목록 조회 → 등록/수정/삭제
   │       (editor: 본인 등록 콘텐츠만 수정/삭제)
   ├── [super_admin, content_admin] 회사소개 관리 페이지
   │     → SafeInsight + In-Guide 슬라이드 on/off 토글
   ├── [super_admin, content_admin] 동영상 관리 페이지
   │     → 동영상 등록/삭제
   ├── [super_admin, content_admin] 이미지 관리 페이지
   │     → 이미지 등록/삭제
   └── [super_admin] 사용자 관리 페이지
         → 계정 목록 조회 → 역할 변경/계정 비활성화

[디스플레이 화면 /]

1. TV/키오스크에서 / 진입 (로그인 불필요, 공개)
   ↓ 자동 슬라이드쇼 시작

2. Swiper 기반 풀스크린 자동재생 슬라이드쇼
   ├── 조직도 슬라이드 (항상 표시)
   ├── 뉴스 슬라이드 (활성 뉴스 있을 때)
   ├── 방문자 슬라이드 (활성 방문자 있을 때)
   ├── 회사소개 슬라이드 (is_enabled=true일 때)
   ├── 이미지 슬라이드 (활성 이미지 있을 때)
   └── 동영상 슬라이드 (활성 동영상 있을 때, 풀스크린 재생)
   ↓

3. Supabase Realtime 구독 — 관리자 변경 사항 즉시 반영 (페이지 새로고침 불필요)
```

## 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F001` | **접근:** 비로그인 상태에서 /admin 진입 시 자동 리디렉션

| 항목 | 내용 |
|------|------|
| **역할** | 관리자 앱 진입을 위한 이메일/비밀번호 인증 전용 페이지 |
| **진입 경로** | 비로그인 상태로 /admin 진입 시 자동 리디렉션, 또는 직접 접근 |
| **사용자 행동** | 이메일/비밀번호 입력 후 로그인 버튼 클릭. 회원가입 페이지로 이동하는 링크 클릭 |
| **주요 기능** | · 이메일/비밀번호 입력 폼 (Zod 유효성 검사)<br>· Supabase Auth 기반 인증 처리<br>· 인증 실패 시 에러 메시지 표시<br>· 회원가입 페이지 링크<br>· **로그인** 버튼 |
| **다음 이동** | 성공 → 대시보드 페이지, 실패 → 에러 메시지 표시 (페이지 유지) |

---

### 회원가입 페이지

> **구현 기능:** `F026` | **접근:** 로그인 페이지에서 회원가입 링크 클릭

| 항목 | 내용 |
|------|------|
| **역할** | 신규 시스템 사용자 가입 신청 전용 페이지 |

### 프론트엔드 프레임워크

- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 폼 & 검증

- **react-hook-form 7.80.0** - 폼 상태 관리
- **Zod 4.4.3** - 스키마 검증 라이브러리

- **Zustand 5.0.14** - 클라이언트 UI 상태 관리 (인증 상태 persist 포함)
